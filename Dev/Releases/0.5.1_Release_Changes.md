# Unity Code MCP Server 0.5.1 Release Changes

## TL;DR

- The MCP server now registers reliably in Claude/Codex **every session, even if Unity is not open yet** — no more "check the connection each time."

## 🐞 Fix

Previously the bridge forwarded the capability-listing calls (`tools/list`, `prompts/list`, `resources/list`) to Unity using the full 180-second request timeout. These run during the MCP **handshake**, so if Unity was not open (or was mid-reload) when the client started, the handshake hung and the client marked the server as failed — it only appeared if Unity happened to be running first.

Now:

- The listing calls use a short **8-second** timeout, so the handshake always completes quickly regardless of Unity state and the server registers every session.
- The tool list is cached to `.unityCodeMcpServer/tools-cache.json` on every successful list and served from that cache when Unity is unavailable, so tools still appear (and become callable as soon as Unity comes up) without restarting the client.
- Actual tool **calls** still use the full request timeout and return a clean error if Unity is down.

No configuration changes required. Update the package to pick it up.
