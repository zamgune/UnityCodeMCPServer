# Unity Code MCP Server 0.5 Release Changes

> This is the first release of the **zamgune/UnityCodeMCPServer** fork, built on upstream 0.4.

## TL;DR

- Requests no longer hang when the editor is unfocused — including during Play Mode — so agents running in a terminal (codex, Claude Code) work without babysitting editor focus.
- New `building-unity-ui-from-html` skill converts HTML/CSS UI prototypes into faithful uGUI.
- Installs cleanly in projects that already vendor UniTask, and no longer collides with the official `com.unity.editorcoroutines` package.

## ✨ What's New for You?

The 0.5 release focuses on making agent-driven workflows reliable when the Unity editor is in the background, plus a new UI-building skill.

- **No more manual editor refresh:** A throttled polling fallback recovers requests the `FileSystemWatcher` misses (frequent on macOS), and incoming requests auto-refresh the AssetDatabase, so externally edited scripts compile without the editor needing focus.
- **Play Mode works while unfocused:** Entering Play Mode now enables `Application.runInBackground` for the session, and message-file I/O runs synchronously on the main thread — so requests issued from an unfocused editor are serviced in ~0.2s instead of hanging until you click back into Unity.
- **Build UI from an HTML mockup:** The new `building-unity-ui-from-html` skill gives an agent a full workflow (spec extraction → chunked build → numeric rect-dump + screenshot verification) for reproducing an HTML/CSS prototype as uGUI.
- **Smoother installs:** The package no longer hard-depends on the `com.cysharp.unitask` registry package (any UniTask in the project satisfies the by-name asmdef reference), and an unused vendored EditorCoroutines DLL that clashed with the official package was removed.

## ⚠️ Upgrading

- **Reliability toggles default to on, but pre-existing settings assets need a check.** A settings asset created before these fields existed deserializes the new booleans to `false`, silently disabling them. Open **Tools/UnityCodeMcpServer/Show or Create Settings** and confirm **Auto Refresh Assets On Request** and **Run In Background During Play Mode** are enabled (or add `AutoRefreshAssetsOnRequest: 1` and `RunInBackgroundDuringPlayMode: 1` to the `.asset` directly). Fresh installs are unaffected.
- **Running two projects at once:** give each project's MCP server a unique name in your client config (e.g. `unity-projectA` / `unity-projectB`) and run a separate agent session per project. The transport itself is fully project-isolated.

## 🚀 Key Improvements

### 🖥 Background / unfocused reliability

- **Polling fallback:** a throttled `EditorApplication.update` rescan recovers requests missed by the `FileSystemWatcher`.
- **Auto AssetDatabase refresh:** incoming requests refresh assets so externally edited scripts compile without editor focus (`AutoRefreshAssetsOnRequest`, default on).
- **Compile-aware deferral:** requests that arrive mid-compile are deferred and processed against the fresh assemblies after the reload, instead of being rejected.
- **Run-in-background keeper:** Play Mode enables `Application.runInBackground` for the session so the editor keeps ticking while unfocused (`RunInBackgroundDuringPlayMode`, default on; runtime-only, no PlayerSettings/build impact).
- **Synchronous message I/O:** message files are read/written synchronously on the main thread, avoiding thread-pool continuations that macOS App Nap stalls when the editor is backgrounded.
- **Stale-file cleanup:** orphaned `.tmp`/request/response files from interrupted reloads or crashed bridges are swept on server start.

### 🎨 New skill

- **`building-unity-ui-from-html`:** HTML/CSS → uGUI workflow with a CSS-to-uGUI mapping reference and verified helper snippets (rect dump, bordered-panel pattern, language-capable TMP font check).

### 📦 Packaging / compatibility

- **Vendored UniTask support:** dropped the `com.cysharp.unitask` UPM dependency; any UniTask copy in the project satisfies the asmdef reference.
- **`--project-root` bridge option:** run the STDIO bridge from outside the Unity project.
- **EditorCoroutines collision removed:** deleted an unused vendored `Unity.EditorCoroutines.Editor.dll` that clashed with `com.unity.editorcoroutines`.
